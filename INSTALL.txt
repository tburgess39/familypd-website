FamilyPD Build 8/9 Clean Least-Privilege Patch

REPLACE these existing Apps Script files:
1. IdentityService.gs
2. IdentityDocumentService.gs
3. ReviewService.gs
4. ReviewDocumentService.gs

ADD this new Apps Script file:
5. FPDDriveFileService.gs

Do not replace IdentityClient.html or ReviewClient.html.

MANIFEST
In appsscript.json, use this Drive scope:
https://www.googleapis.com/auth/drive.file

Remove this broader scope if present:
https://www.googleapis.com/auth/drive

ADVANCED SERVICE
In Apps Script, open Services (+), add Drive API, and use version v3.
This is the Advanced Drive service used by the clean adapter.

WHY THIS PATCH
- Restores IdentityService.getWorkspaceView().
- Keeps the Family Identity module under FPDFamilyIdentityServiceV90.
- Removes every DriveApp call introduced by Builds 8 and 9.
- Uses stored workspace IDs instead of searching the user's whole Drive.
- Keeps Progress Review and Identity Doc/PDF generation available through drive.file.

DEPLOY
Save all files, run fpdIdentityGetData once, then deploy a new web-app version.
