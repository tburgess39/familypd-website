FAMILYPD BUILD 9.0 — FAMILY IDENTITY & VALUES CENTER
====================================================

ADD THESE THREE NEW FILES TO GOOGLE APPS SCRIPT
-----------------------------------------------
1. IdentityService.gs
2. IdentityDocumentService.gs
3. IdentityClient.html

Do not overwrite the working Code.gs, Config.gs, MeetingService.gs,
WorkspaceService.gs, DocumentService.gs, ReviewService.gs,
ReviewDocumentService.gs, ReviewClient.html, Styles.html, or Client.html.

ONE SMALL INDEX.HTML CHANGE
---------------------------
In Index.html, add this line immediately before the closing </body> tag:

<?!= include('IdentityClient'); ?>

Place it after the existing ReviewClient include line, so the bottom of the file
contains both lines in this order:

<?!= include('ReviewClient'); ?>
<?!= include('IdentityClient'); ?>
</body>

PERMISSIONS
-----------
No new permission change is required if Build 8 is already working and
appsscript.json now uses:

https://www.googleapis.com/auth/drive

DEPLOY
------
1. Save all files.
2. Deploy > Manage deployments.
3. Edit the current web app deployment.
4. Select New version and deploy.
5. Refresh the app with Ctrl+Shift+R.

WHAT THIS BUILD ADDS
--------------------
- A Family Identity button in the existing navigation when possible.
- Family/household name, tagline, story, and desired home environment.
- Family mission and future-facing vision.
- Suggested and custom core values with editable definitions and behaviors.
- Family members, relationships, roles, responsibilities, and strengths.
- Family strengths, traditions, culture/heritage notes, and commitments.
- Editable Google Doc plus PDF generation.
- Files saved in 01 Family Profile & Values.
- Generated files automatically visible in Build 8's Progress Review Center.
- Per-user saved data and five-minute caching.
- Versioned global names and unique server endpoints to prevent collisions.

PRIVACY
-------
The interface and generated plan remind families not to enter passwords,
account numbers, Social Security numbers, exact dates of birth, medical record
information, or other sensitive data.
