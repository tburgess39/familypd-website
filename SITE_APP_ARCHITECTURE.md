# FamilyPD Website and App Architecture

## Public website — familypd.org

Purpose: explore, learn, understand, test examples, and find public resources.

Public website features:
- FamilyPD overview and framework
- Five pillars and four guiding principles
- Public Resource Hub
- Learning Tools & Project Resources
- Kids & Teens Hub
- Youth Creator Lab
- Sample or practice versions of FamilyPD tools
- Programs, impact, partnerships, and community information

The public website is not the permanent private family record system.

## Private app — app.familypd.org

Purpose: implement FamilyPD inside the household.

Planned app features:
- Family profile
- Mission, vision, values, and roles
- Real family meeting planner and records
- Goals, checkpoints, and progress
- Household policies, systems, and responsibilities
- Google Drive workspace creation
- Editable Google Docs and PDFs
- Archive and managed file controls
- School and community opportunity connections

## School and community bridge

Schools and community organizations can:
- share public learning resources;
- teach skills required for household projects;
- host FamilyPD workshops;
- review youth projects;
- connect families with CTE, careers, tutoring, libraries, workforce programs, health resources, and other support;
- provide mentors, facilitators, and subject-matter expertise.

## Privacy boundary

The public website does not connect GitHub to a family’s private Google Drive.

Google permissions, private Drive files, PDFs, and implementation records belong in the FamilyPD App.
